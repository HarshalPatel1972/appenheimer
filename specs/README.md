# Specs Index

This directory contains technical specifications for Appenheimer features.

## Terminology and Differences

- **Specs:** Define exact behavior. These detail the concrete inputs, outputs, edge cases, and expected behavior of a specific feature or API.
- **Docs (in /docs):** Define philosophy and high-level architecture. They explain the "why" and "how" at a conceptual level.
- **RFCs (in /docs/rfc):** Request for Comments. These define *proposed* features that are still open for debate and have not yet been approved.
- **ADRs (in /docs/adr):** Architecture Decision Records. These are historical records of finalized architectural decisions.
