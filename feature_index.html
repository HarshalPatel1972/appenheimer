Appenheimer Feature Index

✅ Confirmed Features

Feature #1: Intent Search (non-AI)

A deterministic, fast, free, and reliable search engine using synonym dictionaries, aliases, stemming, fuzzy search, and per-app tags/keywords instead of LLMs.

Feature #2: Capability Tags

Categorical tags assigned to apps to ensure deterministic sorting, filtering, and discovery without relying on natural language processing.

Feature #3: Hover Preview

A fast, at-a-glance tooltip showing 3–5 core capabilities when hovering an app card, while smoothly blurring the rest of the canvas to maintain focus.

Feature #4: App Details Panel

A fact-sheet modal or side panel displaying platforms, pricing, key features, and screenshots, allowing users to close it and seamlessly return to their un-lost spatial graph.

Feature #5: Live Context-Aware Filters

"Discover first. Filter later." Filters only appear after a search is executed to prevent users from prematurely hiding premium apps. Filters are dynamically generated based on the search context (e.g., "Templates" for design, "4K" for video) and instantly sort/hide results without a page reload.

Feature #6: Curated Collections (Stacks)

Expert-curated stacks of apps (e.g., "Best for Students", "Designer Starter Pack") to help beginners discover tools. A few top collections are surfaced on the homepage, with a "View All" page hosting the full library.

Feature #7: Saved Apps

Users can explicitly bookmark apps they like ("Saved Apps" or "Your Collection") using local storage, without requiring an account.

Feature #8: Search-Specific "Last Used" Tag

A subtle memory cue badge ("Last Used") shown on the app card if the user previously launched it from that specific search query (e.g. Canva for "Design"). No dedicated "Recent" section, just context-aware tags.

Feature #9: Direct Routing via Link Icon

Clicking an app card opens the App Details panel. A specific external link icon on the card routes the user directly to the official website (no intermediate "App Journey" page).

Feature #10: Curated Meaningful Badges

Rare, specific badges (e.g., "Editor's Choice", "Beginner Friendly", "Industry Standard") applied sparingly to help users avoid sketchy tools and quickly identify the best options.

Feature #12: Hidden Gems

A toggle that surfaces lesser‑known but excellent apps (e.g., Kittl, Photopea, Affine) to help users discover tools they wouldn't otherwise find.

Feature #13: Keyboard Navigation

Full keyboard support: arrow keys to move between apps, Enter to open, Esc to return to search, enabling a fast, desktop‑like experience.

Feature #18: Keyboard Shortcut Cheat Sheet

Press ? to show an overlay with shortcuts (e.g., / focus search, Esc back, arrows navigate, Enter open). Hidden UI element that powers up power users.

Feature #22: Search Anywhere (Universal Search)

One search bar, zero modes. Users can type app names, tasks, categories, features, platforms, or combos (e.g., "free markdown notes for students"). The deterministic engine resolves context without dropdowns.

Feature #23: Zero Results (Friendly Fallback)

If no exact match is found, show a friendly message with closest matches instead of a dead‑end “No results”. Keeps the experience alive.

Feature #24: Instant Search (Animated Intent Preview)

While typing, floating keyword bubbles animate across the canvas, refining as the user types. Users can click a suggestion to instantly show its apps, or press Enter to search the exact query. This creates anticipation and confirms intent.

Feature #25: Search Chips

After results appear, the original query becomes removable chips (e.g., Free · Windows · Video Editor). Removing a chip instantly updates results.

Feature #26: "What's New" Badges on App Cards

When an app receives a major update, a tiny badge (✨ AI Removed Background, 🆕 Offline Mode, 🚀 New Mobile App) appears, earned only after the user has visited that app.

Feature #28: Favorites in Results

Saved apps float to the top of future searches with a ⭐ indicator, providing quick access.

Feature #29: Search Sessions

The site remembers the last search state; a subtle pill offers "Continue" or "New" to resume or start fresh.

Feature #33: Feedback Button

A tiny ⚑ icon on each app card lets users report broken links, wrong information, or suggest new apps with a single click, keeping the directory up‑to‑date.

Feature #34: Changelog

A simple page lists recent site updates (new apps, improved search, bug fixes) to show users the project is actively maintained.

Feature #35: App Status Badges

Each app card shows a small status badge (✅ Active, ⚠️ Discontinued, 🔄 Acquired, 🧪 Beta) so users avoid dead or transitional software.

Feature #36: Trust Signals (Phase 2)

Future feature: Shows factual indicators like "50M+ users", "Since 2012", "Company-backed" to help users judge credibility in 2 seconds.

Feature #37: Developer Claimed Page

Phase 1: Developers can claim their page and suggest updates to description, pricing, etc., which go into a review queue. Phase 2: Verified developers can update instantly.

Feature #38: Request a Search

If a search has no results, users can click "Request this search" to help prioritize what to add based on real demand.

Feature #40: Community Corrections

Users can suggest edits to incorrect information (like Wikipedia) which go into a moderation queue, separate from just reporting a bug.

Feature #42: Service Status (Future)

If an app is down (and status data is available via API), it renders greyed out. Hovering displays "This service is currently experiencing an outage." Avoids active pinging to maintain leanness.

Feature #44: Anonymous Session Analytics

Developer-only admin feature tracking anonymous session flow (search queries, latency, hovers, clicks, time spent) as a timeline to help prioritize improvements and debug issues without collecting personal data.

Feature #52: Smart Redirect

Clicking the link icon on an app card opens the official website immediately in a new tab, while a subtle, temporary toast on the current site confirms "✨ You're now exploring [App]."

Feature #53: App Popularity Over Time

Shows dynamic trends based on internal ecosystem data (e.g., 🔥 Trending This Week, 📈 Growing Fast, 👑 Most Clicked) instead of static external rankings.

⏸ Parked Features

Feature #11: Official Website Indicator

Shows a tiny badge (✅ Official) on apps that exist on multiple sites, indicating the link leads to the real website.

Feature #17: Search History

A tiny clock icon in the search bar opens a list of the last 5–10 searches, allowing one‑click re‑run. Parked.

Feature #12: "Why People Use It" Hover Details

On hover, displays real use cases (e.g., for Canva: Instagram posts, presentations, resumes, thumbnails, posters) instead of generic marketing features.

Feature #41: App Timeline

On click, shows major milestones (launch, mobile app, AI features) to indicate app maturity. Parked for v1.

Feature #43: App Collections by Users

Users can create and share collections of apps (e.g., "My College Toolkit"). Parked as a potential future community feature.

Feature #45: Search Insights

After a search, show one tiny context-aware statistic (e.g., "Showing 127 apps", "42 apps are free") to make the experience richer. Parked.

Feature #48: App Availability History

Show reliability over time (e.g., 99.98% uptime). Parked for V2.

Feature #50: Compare Search Results Across Time

Highlight what changed since the user last searched for a query (new apps, discontinued apps). Parked for the future.

Feature #51: App Playground

Show an interactive screenshot/mockup of the app interface on hover or click so users know what it looks like before visiting. Parked for v2.

❌ Killed Features

Feature: "Recently Added" Badge

Badge indicating newly added apps was deemed too deep; not included to keep interactions under a minute.

Feature #39: App Relationships

Killed because showing "Works well with", "Imports from", etc. is content that should be displayed in the app details view, not a new feature.

Feature #46: Smart Empty State

Killed because the beautiful landing page design will handle the initial 2-5 seconds of the website visit organically. Not a programmatic feature.

Feature #47: Keyboard Command Palette (Ctrl+K)

Killed because it is a duplicate of the already confirmed keyboard shortcuts cheat sheet and universal search capabilities.

Feature #54: Search Completion Rate

Killed as a standalone feature. This is naturally covered by the Anonymous Session Analytics (Feature #44) tracking flow and success states.

⏳ Pending Decision

Feature: Copy Link to Search

After searching, the URL reflects the query (e.g., /search/design-app) for sharing/bookmarking. Decision pending.

⚙️ Search Engine Requirements

Alias Resolution & Duplicate Detection (Formerly Feature #49)

Not a UI feature. The search engine backend must handle canonical app IDs, aliases (e.g., "vscode" → "Visual Studio Code"), synonyms, and fuzzy matching to prevent duplicates in results.
