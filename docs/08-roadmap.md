# Roadmap

> Build a strong foundation first. Features come later.

---

# Purpose

This document defines the long-term development roadmap of Appenheimer.

It is intended to provide direction rather than fixed deadlines.

The roadmap may evolve as the project matures.

---

# Guiding Principles

- Foundation before features.
- Quality over speed.
- Finish before expanding.
- Build for the long term.
- Never sacrifice maintainability for rapid development.

---

# Phase 0 — Foundation

Goal:

Prepare the project for long-term development.

Deliverables:

- Repository structure
- Documentation
- Engineering principles
- Architecture
- RFC process
- Development workflow

Status:

Completed

---

# Phase 1 — Core Infrastructure

Goal:

Build the technical foundation.

Deliverables:

- Backend foundation
- Frontend foundation
- Database
- Search engine
- Authentication
- Admin foundation
- CI/CD
- Monitoring

Success Criteria:

A stable platform capable of supporting feature development.

---

# Phase 2 — Core Product

Goal:

Deliver the primary user experience.

Deliverables:

- Search
- Floating canvas
- Application details
- Hover interactions
- Search suggestions
- Filters
- Favorites
- Collections
- Continue session

Success Criteria:

Users can successfully discover and explore software.

---

# Phase 3 — Community

Goal:

Allow the community to improve Appenheimer.

Deliverables:

- Feedback
- Suggest application
- Suggest edits
- Report issues
- Developer claims

Success Criteria:

The platform continuously improves through user contributions.

---

# Phase 4 — Intelligence Through Data

Goal:

Improve discovery using platform insights.

Deliverables:

- Better ranking
- Better synonym handling
- Better analytics
- Better recommendations (deterministic)
- Search quality improvements

Success Criteria:

Discovery becomes more accurate without compromising determinism.

---

# Phase 5 — Growth

Goal:

Scale the platform while preserving quality.

Deliverables:

- Performance improvements
- Operational tooling
- Better moderation
- Better data quality
- Infrastructure scaling

Success Criteria:

The platform scales without sacrificing user experience or maintainability.

---

# Continuous Work

The following continue throughout the lifetime of the project:

- Bug fixes
- Performance improvements
- Documentation
- Refactoring
- Testing
- Security
- Accessibility

---

# Final Principle

> Appenheimer grows by strengthening its foundation, not by accumulating features.
