# Non-Goals

> Appenheimer becomes better by saying "no" to unnecessary complexity.

---

# Purpose

This document defines what Appenheimer intentionally does not aim to become.

Any proposal that conflicts with these principles should be carefully reconsidered.

---

# Product Non-Goals

Appenheimer is NOT:

- an AI product
- a chatbot
- a SaaS dashboard
- a marketplace
- an app store
- a software download website
- a review website
- a social network
- a forum
- a blogging platform
- a news website

Its purpose is software discovery.

Nothing more.

---

# Engineering Non-Goals

The project will not:

- rely on AI in production
- depend on proprietary AI services
- overengineer simple problems
- optimize before measuring
- introduce unnecessary dependencies
- sacrifice maintainability for short-term speed
- tightly couple components

---

# User Experience Non-Goals

The product will not:

- require users to create an account to search
- force onboarding before searching
- interrupt users with tutorials
- overwhelm users with information
- hide useful software behind unnecessary filters
- require multiple pages to understand a product
- waste the user's time

---

# Business Non-Goals

The platform will not:

- sell search rankings
- disguise advertisements as recommendations
- manipulate rankings for commercial reasons
- prioritize revenue over user trust

Trust is more valuable than short-term monetization.

---

# Design Non-Goals

The interface will not:

- prioritize visual effects over usability
- use animations without purpose
- add features simply because competitors have them
- become cluttered over time

Every visual element must contribute to the user's goal.

---

# Scope Control

Before adding any new feature, ask:

- Does it help users discover software faster?
- Does it respect the user's time?
- Is it simple?
- Is it maintainable?

If the answer is "no", the feature should probably not exist.

---

# Final Principle

> Every new feature increases complexity.

Complexity should only be accepted when it creates meaningful value for users.
